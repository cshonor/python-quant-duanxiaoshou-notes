# A股多因子入门：每日行动清单 + 避坑检查清单

> 适用对象：新手从 0 到 1 做 A 股多因子。  
> 使用方式：每天按清单推进；每周做一次复盘；每次回测前后都跑一遍避坑检查。

---

## 一、每日行动清单（可打卡）

### Day 1：数据与宽表底座
- [ ] 确认数据源（Tushare / JQData / AkShare）与权限范围。
- [ ] 建立双主键：`trade_date + ts_code`（或等价代码字段）。
- [ ] 拉取基础表：行情、复权因子、行业、市值、交易状态。
- [ ] 初步过滤股票池：ST、退市、上市未满 6 个月、长期停牌标的。
- [ ] 保存为统一宽表（建议 Parquet/Feather）。

### Day 2：标签与对齐
- [ ] 构造未来收益标签：`ret_1d`、`ret_5d`、`ret_10d`（和调仓周期一致）。
- [ ] 确认标签是“未来收益”，因子是“当期可得信息”（时间严格对齐）。
- [ ] 补齐交易日历，处理停牌与缺失值（优先标记再填充）。
- [ ] 输出一份“标签统计摘要”（均值、标准差、分位数）。

### Day 3：基础因子池（先少后多）
- [ ] 先做 5~10 个基础因子：动量、波动率、换手、量比、估值等。
- [ ] 因子计算后做完整性检查：缺失率、异常值比例、极端分布。
- [ ] 因子统一命名规范（如 `f_mom_20d`、`f_vol_20d`）。
- [ ] 保存因子表并写明计算口径。

### Day 4：预处理与中性化
- [ ] 每日横截面去极值（3σ 或 MAD）。
- [ ] 每日横截面标准化（Z-score 或 Rank）。
- [ ] 做市值中性化、行业中性化（回归残差或分组去均值）。
- [ ] 记录处理前后分布变化，防止过度处理。

### Day 5：单因子检验
- [ ] 计算 IC（Spearman 推荐）与 IC 均值、IC 胜率、ICIR。
- [ ] 做分层回测（至少 5 组）：检查单调性与首尾组差异。
- [ ] 做衰减测试：T+1 / T+2 / T+5 的 IC。
- [ ] 淘汰弱因子：IC 近 0、胜率低、分层不单调。

### Day 6：多因子合成与约束
- [ ] 因子相关性矩阵检查，去掉高度相关（如 > 0.8）因子。
- [ ] 先用简单可解释权重：等权 / IC 均值权重 / ICIR 权重。
- [ ] 输出综合评分 `score` 并生成选股列表（如 top N）。
- [ ] 加仓位约束：单票上限、行业上限、市值风格偏离上限。

### Day 7：回测、复盘与迭代
- [ ] 设置真实交易参数：手续费、滑点、最小成交单位、涨跌停约束。
- [ ] 回测输出核心指标：年化、回撤、夏普、卡玛、胜率、盈亏比。
- [ ] 样本内 / 样本外分段验证，检查风格切换期表现。
- [ ] 写复盘结论：保留因子、淘汰因子、下周优化计划。

## 二、每次回测前的避坑检查（必须全过）

### A. 时间与数据泄露
- [ ] 因子是否只用到当日及以前数据。
- [ ] 标签是否是未来收益，且没有反向泄露到因子。
- [ ] 标准化/中性化是否按“每日横截面”做，而非全样本全局做。

### B. 交易可实现性
- [ ] 是否排除无法成交场景（停牌、涨跌停、流动性不足）。
- [ ] 是否设置了合理手续费、滑点、冲击成本。
- [ ] 调仓频率是否与标签预测周期一致。

### C. 因子有效性与稳健性
- [ ] IC 均值是否显著为正，IC 胜率是否稳定。
- [ ] 分层是否单调，首尾组收益差是否持续。
- [ ] 因子衰减曲线是否符合预期（不是只有 T+1 有效）。

### D. 过拟合与鲁棒性
- [ ] 是否做样本外验证，且样本外没有明显崩塌。
- [ ] 参数数量是否过多（避免“为拟合而拟合”）。
- [ ] 牛/熊/震荡分市场环境是否都测试过。

## 四、最小可行版本（MVP）建议

如果你刚开始，先做这个最小闭环：
- 5 个基础因子（动量、波动率、换手、成交额变化、估值）
- 每周调仓一次
- top 20 等权持仓
- 回测 3 年（含至少一段下跌市）
- 每周只优化 1 个模块（因子、权重、风控三选一）

这样最容易快速走通“可运行 + 可评估 + 可迭代”的闭环。
